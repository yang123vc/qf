Help
  1. Please note that you can't use the following names for your fields: id, created, created_by, modified
  2. You can't use any SQL keyword
  3. Please note that the first three fields would be in the list view of the documents.
  4. Please note that files, section break and tables are excluded from list view
  5. All fields which are not text or table or section break will have indexes applied to them.
  6. For select fields, put the list of options each separated by a newline in the Other Options textarea.
  7. For link fields (foreign keys between QF Documents), put a QF document structure name in the Other options textarea
  8. For table fields, put a QF document structure in the Other Options textarea that has being created with child_table option selected
  9. For file fields if you want to accept only some file types: enter the file type(s) (without dot) and add it to the Other Options textarea. The file types must be separated by newlines.
  10. Child table types must not have files, section break or tables.
  11. Every field that is not a text, table, file is indexed
  12. ReadOnly / Calculated option has no effect on tables and files
